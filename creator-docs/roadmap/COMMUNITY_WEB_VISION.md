# 社区站愿景：网页端 · 三板块（论坛 / 角色包 / 插件）

本文记录 **产品层** 设想，与 [MARKET_LAUNCHER_INTEGRATION.md](./MARKET_LAUNCHER_INTEGRATION.md)（发版与市场联动、启动器入口）配合使用：前者偏 **分发与发现**，本文偏 **社区形态与内容分区**。

---

## 1. 总体形态

- **主载体**：**网页**（桌面启动器提供入口，打开浏览器或后续内嵌 WebView）。
- **三大板块**（建议主导航一级就分三块，避免混在一起）：

| 板块 | 核心内容 | 与 oclive 技术的关系 |
|------|----------|----------------------|
| **论坛** | 类 **百度贴吧** 的版块/thread、楼中讨论；可选：用户上传 **聊天记录**（节选/脱敏），帖子下 **评论** | 不替代运行时；需注意 **隐私与同意**（见 §4） |
| **角色包** | 类 **C 站** 的 **展示与发现**：立绘/截图、配置说明、**如何触发特定剧情**、版本与下载 | 分发 **`.zip` / `.ocpak`**，契约见 [PACK_VERSIONING.md](../role-pack/PACK_VERSIONING.md) |
| **插件** | **侧车 / Remote** 相关：文档、示例链接、讨论区（不是在市场里装 `.dll`） | 与 [REMOTE_PLUGIN_PROTOCOL.md](../plugin-and-architecture/REMOTE_PLUGIN_PROTOCOL.md)、环境变量一致 |

---

## 2. 各板块可落地的功能（设计时可拆用户故事）

### 2.1 论坛（贴吧式）

- 版块划分（示例）：综合、求助、反馈、创作交流、**剧情与攻略**（与「触发条件」内容可互链）。
- 帖子类型：普通帖；**聊天记录分享帖**（附件或代码块 + 可选「关联角色包」）。
- **评论楼**：一级/二级即可，复杂度可控。

### 2.2 角色包（C 站式展示栏）

- **列表 + 筛选**：标签、兼容 oclive 版本、`schema_version`、题材等。
- **详情页**：立绘/截图轮播、作者、**配置要点**（manifest/settings 关键项说明）、**剧情触发说明**（与游戏内 `scene`/事件链对应，以文档为准）。
- **下载**：指向对象存储或 GitHub Release；校验可选 `sha256`。
- 与 [MARKET_LAUNCHER_INTEGRATION.md](./MARKET_LAUNCHER_INTEGRATION.md) 中的 **catalog / 静态索引** 可合并为同一站点的「数据层」，或论坛与目录分服务、域名统一入口。

### 2.3 插件区

- 明确 **插件 = Remote HTTP 侧车**（或将来官方列出的二进制工具），**不是**宿主内动态插件。
- 内容：侧车 README 摘要、环境变量示例、与角色包 `plugin_backends` 的搭配说明、**讨论帖链接**。
- 重度实现仍建议链到 **仓库 / Release**，网站做 **索引与发现**。

## 3. Discord 要不要开？

| 选项 | 优点 | 代价 |
|------|------|------|
| **只做国内论坛（网页）** | 与「贴吧」体验一致、**中文搜索与运营**集中；合规与备案路径相对清晰（视主体与内容而定） | 需自建或选型论坛产品（开源 / SaaS） |
| **同时开 Discord** | 国际用户、与 GitHub 生态近、语音与实时性 | **运营分流**、中英文两套规范、审核与社区规则分开 |
| **仅 Discord** | 省自建论坛 | 国内访问与体验不稳定，**不适合作为「主站」** |

**建议**：若以 **国内用户为主**，**主社区仍以网页论坛为主**；Discord 可作为 **可选国际频道**（链接放在页脚），不把核心攻略只放在 Discord。

---

## 4. 聊天记录上传与评论（必考虑的边界）

- **隐私**：默认要求用户 **自行脱敏**（人名、ID、地理位置）；可提供上传前提示与格式建议。
- **同意**：若涉及他人对话，产品侧需提示 **仅上传自己有权的记录**。
- **存储**：文本可存站；大文件走对象存储；**可设大小与频率限制**防滥用。
- **与角色包关系**：帖子可 **关联角色包 ID**（站内条目），便于「同一包下的剧情讨论」聚合。

---

## 5. 与启动器、发版的关系

- 启动器 **「社区」或分入口**（论坛 / 市场 / 插件）→ 打开 **同一主域名** 下路径，例如：  
  `/forum`、`/packs`、`/plugins`。
- **发版同发**：站点与桌面端大版本叙事一致即可（详见 MARKET 文档）；论坛与角色包索引的 **部署节奏** 可与 Release Checklist 挂钩。
