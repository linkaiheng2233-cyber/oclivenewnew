# 开源协议策略（主程序与插件）

本文定义当前阶段的最小协议策略，保证发布前可执行、后续可扩展。

---

## 1) 主程序仓库（oclivenewnew）

- 当前协议：**MIT**（见仓库根 `LICENSE`）。
- 对外文档应明确写出：`Licensed under MIT`.
- 版本发布前，检查 `LICENSE` 文件存在且内容未损坏。

---

## 2) 官方插件（后续新增）

默认要求：

1. 每个官方插件仓库/目录都应有独立 `LICENSE`（默认 MIT）。
2. README 顶部建议声明协议（如 `Licensed under MIT`）。
3. 发布前清单中增加“协议文件存在性检查”。

> 当前脚手架 `npm run scaffold:ui-plugin` 已默认生成 MIT `LICENSE`，用于减少漏项。

---

## 3) 社区第三方插件

- 允许作者自选协议（MIT / Apache-2.0 / GPL 等）。
- 市场页先做“协议展示与提醒”，不做复杂法律校验。
- 若协议缺失，市场侧应给出“未声明协议”提示。

---

## 4) 与安全规则的阶段关系

- 本阶段：先落实协议标注与发布检查（低成本、可立即执行）。
- 下一阶段：再推进安全规则（权限分级、签名/来源校验、沙箱策略等）。

---

## 5) 打包前最小检查项（协议部分）

1. 主仓库存在根 `LICENSE`。
2. 本次新增/更新的官方插件包含 `LICENSE`。
3. 文档或市场信息中可见协议声明。

可用命令（本仓库根目录）：

```bash
npm run check:license
```

该命令会检查主仓 `LICENSE` 与 `mumu` 默认官方插件的 `LICENSE` 存在性。
